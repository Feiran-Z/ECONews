# GitHub Pages Setup Configurator Memory

## Project Patterns

### Repository Structure
- IB Economics companion sites follow a consistent structure: main landing page with About, News, Contact sections
- Two-column layout is common: left scrollable content, right fixed navigation
- Blog post system with pagination (5 posts per page) is standard requirement
- File organization: css/, js/, blogposts/, images/ directories

### Design Specifications
- Main color consistently #880112 (dark red)
- Main font: Manrope, with Inter as alternate sans-serif
- Modern, simplistic theme preferred
- Responsive design required for all projects

### Technical Implementation
- Pure HTML/CSS/JavaScript (no build process)
- CSS Custom Properties for theming
- Flexbox for two-column layout
- JavaScript pagination for blog posts
- Modal system for full post viewing

### File Creation Patterns
- Always create comprehensive README.md with project details
- Include detailed .gitignore for web projects
- Sample blog posts in blog.js for demonstration
- Contact form with validation
- Smooth scrolling navigation

### Common Requirements
- Weekly blog post updates
- IB Economics theory integration
- Original article links required
- Tag system for categorization
- Mobile-responsive design

## Setup Process
1. Create directory structure first
2. Build HTML skeleton with two-column layout
3. Add CSS with theme variables
4. Implement JavaScript functionality
5. Add sample content for demonstration
6. Create documentation files

## Notes
- Users prefer clear separation of concerns (HTML/CSS/JS)
- Sample data helps demonstrate functionality
- Documentation should include setup instructions
- Contact forms should reference GitHub repository
